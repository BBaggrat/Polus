# Content QA report

## Проверенный объем

- HIDDEN content pool: 80 записей, включая movement, encounters, return и failed тексты.
- OPEN_PVP encounter pool: 23 события.
- Цепочки: 5.
- Монстры stage 4: 5 обязательных + 1 дополнительное encounter-событие.
- Return/failed тексты: 8.

## Проверки

- Пустых `title` в encounter-шаблонах не найдено.
- У интерактивных событий есть 2–3 choices.
- У choices есть `resultText`.
- `riskLevel` заполнен.
- Rewards/penalties находятся в `PlayerResources` и `hpDelta`.
- HIDDEN-события не запускают принудительный PvP.
- OPEN_PVP-события используют `START_PVP_DUEL` только как явный outcome choice.
- Монстры оформлены как атмосферные text encounters, не как полноценная PvE-боевая система.
- Пользовательские тексты используют "стычка/топь", не продают режим как арену.

## Найденные проблемы

- Полная проверка стиля всех текстов вручную требует отдельного редакторского прохода после первых игроков.
- Некоторые цепочки короткие, по 2 шага. Это допустимо для release 1.0, но мало для долгосрочной retention-системы.

## Исправлено

- В релизном onboarding уточнена разница между скрытым и открытым режимом.
- Telegram `/start` переписан под текущий концепт топи.

## Оставлено на доработку

- Data-driven content catalog.
- Редакторский review всех текстов после сбора первых behavioral logs.
- Более явные outcome-тексты для PvP aftermath после интеграции реального результата стычки.
