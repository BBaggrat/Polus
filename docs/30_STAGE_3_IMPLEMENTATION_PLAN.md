# Этап 3: база и долгосрочная прогрессия

## Цель

Этап добавляет повторяемый цикл:

`исследование → добыча → возвращение → улучшение базы/снаряжения/карты → новое исследование`.

Живой дневник остаётся главным интерфейсом экспедиции, а существующая PvP-стычка не переписывается.

## Изменения

- `progression/*`: модели базы, улучшений, экипировки, карты, in-memory repositories и сервисы.
- `ExplorationModifierService`: единая точка применения бонусов, риска и потерь.
- `ExplorationService` и `EncounterGenerator`: маршруты, фрагменты карты, эффекты предметов, частичная потеря добычи.
- `BaseController`, `EquipmentController`, `MapController`: новые endpoints.
- `duel/index.html`, `app.css`, `exploration.js`: компактная зона подготовки на базе.

## API

- `GET /api/base/state?playerId=...`
- `POST /api/base/upgrades/{upgradeId}/buy`
- `GET /api/equipment?playerId=...`
- `POST /api/equipment/equip`
- `POST /api/equipment/upgrade`
- `GET /api/map?playerId=...`
- `POST /api/map/routes/{routeId}/select`

Все endpoints требуют `X-Session-Token`; переданный `playerId` обязан совпадать с игроком сессии.

## Связь с исследованием

- склад снижает процент потери временной добычи при провале;
- картографический стол повышает шанс фрагмента;
- настил, оберег, броня и инструменты снижают потерю HP;
- сушилка иногда добавляет припас из loot-события;
- выбранный маршрут меняет предпочтительный тип encounter;
- `OPEN_PVP` даёт множитель добычи 1.25 и более высокий базовый риск потери.

## PvP

Текущая боевая математика и очередь сохранены. Вход из `OPEN_PVP` по-прежнему создаёт событие `pvp_encounter_started` и передаёт управление существующему duel flow.

Полная обратная доставка результата дуэли в незавершённую экспедицию отложена: для неё нужен устойчивый correlation id между exploration и duel. До этого момента бонус `OPEN_PVP` применяется к наградам encounter, а не к результату самой дуэли.

## Ограничения

- база, карта и экипировка хранятся в памяти процесса;
- каталог предметов фиксирован;
- нет сложного инвентаря и крафта;
- визуальная карта не строится;
- PvP-броня не меняет старую боевую формулу;
- численные значения требуют настройки по метрикам этапа 4.
