# Smoke checklist этапа 3

## Backend

1. `GET /api/health` возвращает `ok`.
2. Создать browser demo session и сохранить `X-Session-Token`.
3. `GET /api/player/state` возвращает HP и три ресурса.
4. `GET /api/base/state` возвращает шесть улучшений, экипировку и карту.
5. Купить `storage`: списываются два припаса, уровень становится 1.
6. Повторная покупка без ресурсов возвращает понятную ошибку.
7. `GET /api/equipment` возвращает стартовые плащ, оберег и верёвку.
8. `POST /api/equipment/equip` экипирует доступный предмет.
9. `POST /api/equipment/upgrade` без верстака сообщает требование.
10. `GET /api/map` возвращает фрагменты и три маршрута.
11. Закрытый маршрут нельзя выбрать.
12. Исследование стартует в `HIDDEN` и `OPEN_PVP`.
13. В журнале появляются записи о фрагментах и применённых бонусах.
14. При провале теряется только часть добычи; склад снижает процент.
15. Существующая дуэль и очередь продолжают работать.

## Frontend

1. Открыть `/duel` на ширинах 320, 360, 390 и 430 px.
2. На базе видны HP, ресурсы и вкладки подготовки.
3. Вкладки «Улучшения», «Снаряжение», «Карта» переключаются без перезагрузки.
4. Недоступные действия disabled.
5. Покупка обновляет ресурсы и уровень.
6. На карте видны условия закрытых маршрутов.
7. Начать выход, сделать несколько шагов и вернуться.
8. Проверить отсутствие горизонтального скролла и наложений.

## Автоматические проверки

```bash
node --check src/main/resources/static/duel/exploration.js
mvn clean package
```

В логах проверить `base_upgrade_bought`, `equipment_equipped`, `map_fragment_found`, `route_selected`, `upgrade_effect_applied`, `exploration_failed`, `resources_lost`.
