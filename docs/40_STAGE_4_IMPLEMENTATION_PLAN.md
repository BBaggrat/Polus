# Этап 4: план реализации

Цель этапа 4 — перевести основу Stage 3 из системы вылазок и прогрессии в более наполненный релизный контур: больше событий, цепочки, существа как encounter-события, находки, retention markers и smoke-подготовка к release 1.0.

## Что уже было на входе

- Stage 2: живой дневник, `ExplorationState`, `EncounterGenerator`, скрытый и `OPEN_PVP` режимы.
- Stage 3: база, снаряжение, карта, маршруты, модификаторы наград/риска.
- PvP-дуэль остается отдельной существующей системой; Stage 4 не меняет боевую математику.

## Решение

- Контент расширен в `EncounterGenerator` без смены API.
- Цепочки вынесены в `EventChainService`, чтобы обычный encounter-flow остался прежним.
- Находки вынесены в пакет `discovery`, это не инвентарь и не экономика.
- Баланс контента собран в `ContentBalance`.
- Analytics markers добавлены в `AppEventType` и логируются через существующий `AppEventLogger`.
- Фронт `exploration.js` получил вкладку "Находки", marker активной цепочки и типизированные записи дневника.

## Ограничения этапа

- Данные цепочек, находок и analytics debug in-memory.
- Полная интеграция результата PvP-дуэли в дневник пока не делалась; есть минимальный adapter-журнал при старте стычки.
- Backend API и дуэльная математика не менялись.
